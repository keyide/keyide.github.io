关于程序预解析和高阶函数

程序预解析

  • 在js程序执行之前，浏览器会先对程序进行预解析，在预解析时浏览器会先对变量赋值和函数声明进行处理
  • 当浏览器进行预解析时，遇到变量赋值的语句，浏览器会先将变量读取，但不读取赋值，变量的赋值将在执行程序时在进行(变量提示)；
  • 而遇到函数声明时（不是函数表达式），浏览器会将整个函数的声明全部读取完成(函数提升)，在执行程序时，若不进行调用，浏览器将不会再去读取
  • 函数表达式类似于普通变量赋值，在预解析阶段不会读取赋值部分，在执行阶段才进行赋值

高阶函数

  • 所谓的高阶函数就是指，函数可以当作参数进行传递，也当作返回值进行返回

创建函数的方式

函数声明

  • 函数声明有函数提升,在预解析阶段，如果发现有同名的变量和函数声明，则以函数为主，但是到了执行阶段，如果同名变量赋值执行过了，则会把同名的函数声明给覆盖掉了

函数表达式

  • 函数表达式没有函数提升，只有变量提升
  • 根据一些条件来定义生成不同的函数,则这个时候建议使用函数表达式

new Function 创建函数

  • 这是通过 Function 构造函数来创建函数
  • Function 构造函数的最后一个参数是函数体：字符串的形式，最后一个参数前面的所有参数都是形参
  • new Function 的形式会在执行阶段根据参数解析创建函数的执行
  • 它也有一个好处：可以根据字符串来生成函数
  • 根据条件生成不同的函数,字符串的方式控制起来更灵活,做一个高层次的框架设计的时候会用到这个东西

函数的原型链

  • 函数也是对象，函数也有自己的原型，所以也就有原型链

  • 创建一个新的函数，其实就是通过 Function 构造函数生成了一个新的实例，所以所有实例函数的_proto_都指向 Function 的原型对象，Function 自己的_proto_也指向自己的原型对象，也就是说：所有实例函数的_proto_都指向 Function.prototype ,Function.prototype 的_proto_指向 Object.prototype, Object.prototype 的_proto_指向 null,这就是函数的原型链
