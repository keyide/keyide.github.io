字符串方法整理

js
  • 字符串

字符串方法整理 ：
关于字符串方法

slice()

  • 提取字符串中的一部分
  • 返回新的字符串
  • 原字符串不会改变
  • string.slice(indexStart,indexEnd);
    • indexStart 开始提取的索引，indexEnd 提取结束位置(不包含)
    • 若只传一个参数，则表示从该参数位置提取到最后
    • 若为负数，则表示从后向前数的位置，但提取方向还是向右找
    • 若结束位置参数，比开始位置参数还要小，或相等，都返回空字符

split()

  • 通过把字符串分割成子字符串来把一个 String 对象分割成一个字符串数组
  • 返回分割后的字符串数组
  • 原字符串不会改变
  • str.split(separator，limit);
    • separator 指定用来分割字符串的字符（串）
    • 如果 separator 是一个空字符串，则 str 将被转换为由字符串中字符组成的一个数组，数组每一项元素对应字符串的一个字符
    • 如果忽略 separator，则返回整个字符串的数组形式，数组只有一个元素，该元素为整个字符串。
    • 如果传入 separator,当找到一个 seperator 时，separator 会被从字符串中移除，并到移除位置作为一个数组项，返回存进新数组当中的子字符串数组
    • limit 一个整数，限定返回的分割片段数量，split 方法仍然分割每一个匹配的 separator，但是返回的数组只会截取最多 limit 个元素。

substr()

  • 返回字符串中从指定位置开始到指定长度的子字符串
  • 原字符串不会改变
  • str.substr(start, length)；
    • start 开始提取字符的位置。如果为负值,则为从后向前数的位置
    • 如果 start 为正值且大于字符串的长度，则返回空字符
    • 如果 start 为负值且大于字符串的长度，则 start 使用 0 作为开始提取的索引
    • 如果只有一个参数，则以该参数为开始位置，一直截取到最后
    • length 可选参数。一共向后提取的字符数

substring()

  • 返回字符串两个索引之间（或到字符串末尾）的字符串
  • 原字符串不会改变
  • str.substring(indexStart, indexEnd)；
    • indexStart 开始提取字符的位置
    • indexEnd 结束提取字符的位置（不包含）
    • 如果任一参数小于 0 或为 NaN，则被当作 0
    • 如果任一参数大于字符串的长度，则被当作字符串长度执行
    • 如果两个参数相等，则返回空字符串
    • 如果 indexStart 大于 indexEnd，则 substring 的执行效果就像两个参数调换了一样
    • 如果只有一个参数，则会从参数位置一直提取到字符串结束

indexOf()

  • 检测指定值在字符串中首次出现的位置
  • 原字符串不会改变
  • 返回指定值在字符串对象中首次出现的位置，如果不存在，则返回 -1
  • str.indexOf(searchValue, fromIndex)；
    • searchValue 想要查找的字符串
    • fromIndex 可选，如果传入，则表示从该位置向后查找，不传默认从 0
    • 如果 fromIndex 为负数，则按照 0 的方式执行
    • 如果 fromIndex 大于字符串长度，则该方法返回 -1，除非被查找的字符串是一个空字符串，此时返回字符串长度
    • 返回值为指定值在字符串对象中首次出现的位置的索引，从 0 开始

charAt()

  • 返回字符串中指定位置的字符
  • 原字符串不会改变
  • 若果传入参数大于字符串长度，则返回空字符串
  • 其实在高版本浏览器中可以通过下标直接获取指定位置字符串

toLowerCase()

  • 将调用该方法的字符串值转为小写形式，并返回
  • 原字符串不会改变

concat()

  • 将一个或多个字符串拼接起来
  • 返回得到的新字符串
  • 原数组不会改变
  • string.concat(‘hah’);
  • 可以接受多个参数，并按顺序拼接在目标字符串之后

trim()

  • 创建一个字符串的副本，删除前置以及后缀的所有空格，然后返回结果
  • 字符串中间的空格不会改变
  • 原数组不会改变

exec()

  • 检索字符串中的正则表达式的匹配
  • 返回一个数组，其中存放匹配的结果。如果未找到匹配，则返回值为 null
    • 数组的第 0 个元素是与正则表达式相匹配的文本
    • 数组的第 1 个元素是与 RegExpObject 的第 1 个子表达式相匹配的文本（正则表达式中如果有的话）
    • 数组的第 2 个元素是与 RegExpObject 的第 2 个子表达式相匹配的文本（如果有的话），以此类推
    • exec() 方法还返回两个属性。index 属性声明的是匹配文本的第一个字符的位置。input 属性则存放的是被检索的字符串 string(整个参与检索的字符串)
  • RegExpObject.exec(string)
  • 当 RegExpObject 是一个全局正则表达式时，它会在 RegExpObject 的 lastIndex 属性指定的字符处开始检索字符串 string
  • 当 exec() 找到了与表达式相匹配的文本时，在匹配后，它将把 RegExpObject 的 lastIndex 属性设置为匹配文本的最后一个字符的下一个位置
  • 这样就可以通过反复调用 exec() 方法来遍历字符串中的所有匹配文本。当 exec() 再也找不到匹配的文本时，它将返回 null，并把 lastIndex 属性重置为 0
  • 如果在一个字符串中完成了一次模式匹配之后要开始检索新的字符串，就必须手动地把 lastIndex 属性重置为 0
  • 在循环中反复地调用 exec() 方法是唯一一种获得全局模式的完整模式匹配信息的方法

replace()

  • 替换字符串
  • 方法接受两个函数
    • 第一个参数可以是一个 RegExp 对象或者一个字符串（这个字符串不会被转换成正则表达式）
    • 第二个参数可以是一个字符串或者一个函数
  • 如果第一个参数是字符串，那么只会替换第一个子字符串。要想替换所有子字符串，唯一的办法就是提供一个正则表达式，而且要指定全局（g）标志
  • 如果第二个参数是字符串，那么还可以使用一些特殊的字符序列，将正则表达式操作得到的值插入到结果字符串中

  • replace()方法的第二个参数也可以是一个函数
    • 在只有一个匹配项（即与模式匹配的字符串）的情况下，会向这个函数传递 3 个参数: 模式的匹配项、模式匹配项在字符串中的位置和原始字符串
    • 在正则表达式中定义了多个捕获组的情况下，传递给函数的参数依次是模式的匹配项、第一个捕获组的匹配项、第二个捕获组的匹配项……，但最后两个参数仍然分别是模式的匹配项在字符串中的位置和原始字符串。
    • 这个函数应该返回一个字符串，表示应该被替换的匹配项使用函数作为 replace()方法的第二个参数可以实现更加精细的替换操作
  • 这里，我们为插入 HTML 代码定义了函数 htmlEscape()，这个函数能够转义 4 个字符：小于号、大于号、和号以及双引号。实现这种转义的最简单方式，就是使用正则表达式查找这几个字符，然后定义一个能够针对每个匹配的字符返回特定 HTML 实体的函数
